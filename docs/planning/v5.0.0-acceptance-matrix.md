# Officegen-CLI v5.0.0 Acceptance Matrix

Scope: v5.0.0 publish readiness for the native Officegen CLI. This matrix is executable through `npm run v5:acceptance` and records durable evidence under `.officegen/acceptance/v5.0.0/`.

The v5 release keeps GitHub Release native binaries as the primary runtime path. Node is required only for repository development, smoke automation, and legacy tarball compatibility checks.

## A1-A12

| ID | Area | Acceptance requirement | Machine gate |
| --- | --- | --- | --- |
| A1 | Native runtime | The release binary reports the expected version, `runtime.kind=rust-native`, and `nodeRequired=false` in agent capabilities. | `npm run v5:acceptance -- --bin target/release/officegen --expected-version 5.0.0` |
| A2 | Install story | README and installers preserve the `curl`/`irm` GitHub Release installation path, checksum verification, and no-Node runtime claim. | `npm run installer:smoke`; `npm run v5:acceptance` |
| A3 | Agent surface | Agent workflow docs and agent JSON remain scoped to built-in CLI commands and do not advertise external management runtimes. | `npm run v5:acceptance` |
| A4 | Envelope/errors | Success and failure responses in `--agent --strict-json` keep `officegen.envelope@1.2`, redacted paths, capabilities hash, readiness, and stable failure classes. | `npm run v5:acceptance` |
| A5 | Schemas | `schema list`, `schema get`, and invalid/valid `schema validate` are executable and fail closed. | `npm run schema:coverage`; `npm run v5:acceptance` |
| A6 | Render/assets | Render creates non-empty PPTX, DOCX, XLSX, PDF, chart SVG, and diagram SVG artifacts from structured examples. | `npm run v5:acceptance` |
| A7 | Inspect | Inspect works for representative generated PPTX, DOCX, XLSX, and PDF artifacts with structured summaries. | `npm run v5:acceptance` |
| A8 | Edit | Supported DOCX text edit supports dry-run, explicit output, package diff evidence, and no accidental in-place mutation. | `npm run v5:acceptance` |
| A9 | View/verify/diff | SVG view writes artifacts, portable PNG preview fails closed, verify discloses missing native proof, and diff reports the applied text change. | `npm run v5:acceptance` |
| A10 | Safety | Output traversal and unsafe ZIP entries are rejected with structured errors before release. | `cargo test --locked`; `npm run v5:acceptance` |
| A11 | Native assets | Native asset names, checksum files, installer assets, and smoke scripts are wired for all release targets. | `npm run native:assets:check -- --version 5.0.0 --include-installers`; `npm run native:smoke -- --expected-version 5.0.0` |
| A12 | Publish gates | The v5 publish checklist explicitly blocks tagging/release until version, native, Rust, Node compatibility, and pack gates pass. | `npm run release:gate` |
