# Officegen-CLI v4.5.0 Acceptance Matrix

Scope: v4.5.0 native Rust transition release. This matrix turns the v4.5/v5 plan into repo-local acceptance IDs that can be reviewed before tagging. It is a planning and release-gate document, not an implementation status report.

Ownership rule for this workstream: document-only changes under `docs/planning/`, `docs/reviews/`, and optional contract examples under `examples/v45-contract/`.

Status values: `required`, `recommended`, `manual`, `deferred-v5`, `not-applicable`.

## Acceptance Vocabulary

| Prefix | Area | Blocking rule |
| --- | --- | --- |
| `V45-CMD` | Command registry and help surface | Blocks release if agent-visible command metadata, help JSON, and dispatch disagree. |
| `V45-MCP` | MCP exclusion | Blocks release if MCP can run or appear in built-in help/capabilities/release claims. |
| `V45-CAP` | Compact capabilities | Blocks release if agent capabilities are too large, ambiguous, or omit limitations. |
| `V45-ENV` | Envelope, error, schema contract | Blocks release if success/failure JSON is not parseable, stable, and schema-discoverable. |
| `V45-SAFE` | Path and ZIP safety | Blocks release if unsafe output paths or risky ZIPs can be accepted silently. |
| `V45-SMOKE` | End-to-end command smoke | Blocks release if the explicit inspect/render/edit/view/verify/diff/manifest loop cannot be evidenced. |
| `V45-REL` | Release Definition of Done | Blocks release if version, native asset, installer, Rust, Node legacy, and pack gates are incomplete. |

## Minimal Smoke Sequence

The release reviewer may replace fixture paths, but the evidence set should preserve this command shape:

```bash
officegen capabilities --agent --strict-json
officegen help workflow inspect-edit-verify --agent --strict-json
officegen inspect fixtures/sample.pptx --depth summary --agent --strict-json
officegen view fixtures/sample.pptx --format svg --out .officegen/acceptance/v4.5.0/view/sample --agent --strict-json
officegen edit fixtures/sample.pptx --ops examples/v45-contract/v4.5.0-edit-ops.example.json --dry-run --resolve-selectors --agent --strict-json
officegen edit fixtures/sample.pptx --ops examples/v45-contract/v4.5.0-edit-ops.example.json --out .officegen/acceptance/v4.5.0/edit/sample-edited.pptx --agent --strict-json
officegen diff fixtures/sample.pptx .officegen/acceptance/v4.5.0/edit/sample-edited.pptx --visual --agent --strict-json
officegen verify .officegen/acceptance/v4.5.0/edit/sample-edited.pptx --visual --agent --strict-json
officegen manifest inspect .officegen/acceptance/v4.5.0/manifest/manifest.json --agent --strict-json
```

## Release Definition of Done

Before creating or pushing a `v4.5.0` tag:

```bash
npm run version:check
npm run installer:smoke
# macOS/Linux
npm run native:smoke -- --bin target/release/officegen --expected-version 4.5.0
# Windows
npm run native:smoke -- --bin target/release/officegen.exe --expected-version 4.5.0
cargo fmt --check
cargo test --locked
cargo build --release --locked
npm run typecheck
npm test
npm run build
npm run pack:smoke
```

Additional recommended evidence for this release:

- `npm run contract:check`
- `npm run schema:coverage`
- `npm run remediation:check`
- `npm run perfect-spec:evidence`
- `npm run perfect-spec:check -- --gate=publish`
- post-tag native asset and GitHub install smoke for the exact `v4.5.0` release assets

## Deferred to v5

The following are not acceptance blockers for v4.5.0 if they are truthfully disclosed and fail closed:

| v5 item | Reason for deferral | Related v4.5 IDs |
| --- | --- | --- |
| Full workflow runner parity | Rust command shape exists, but unattended workflow execution is still migration work. | V45-CMD-003, V45-SMOKE-007 |
| MCP runtime enablement | v4.5 and v5 keep this outside the built-in CLI; any future adapter belongs outside this runtime. | V45-MCP-001, V45-MCP-002 |
| Native Office/LibreOffice render bridge | Environment-dependent external process support should not be implied by portable runtime smoke. | V45-CAP-003, V45-SMOKE-005 |
| Deep v3.1.x semantic parity | Current Rust runtime is conservative; richer object models and mutations need fixture replay. | V45-SMOKE-001 through V45-SMOKE-006 |
| Broad template/design/layout mutation | Compatibility names can remain while mutation-heavy implementations fail closed. | V45-CMD-003 |
