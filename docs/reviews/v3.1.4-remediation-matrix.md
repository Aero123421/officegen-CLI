## Release Gates

Required before v3.1.4 tagging:

- `npm run version:check`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run pack:smoke`
- `npm run remediation:check`
- `npm run contract:check`
- `npm run schema:coverage`
- `npm audit --audit-level=moderate`
- `npm run security:fixtures`
- `npm run benchmark:local-fixtures`
- `npm run perfect-spec:evidence`
- `npm run perfect-spec:check -- --gate=publish`
- `npm run release-tarball:smoke -- dist/officegen-v3.1.4.tgz`

Post-tag release completion additionally requires fresh v3.1.4 GitHub install and release-tarball smoke evidence.

## Review 1: Agent Contract / False Success

Reviewed response-only JSON budget truncation, runtime envelope objective projection, top-level `partial` semantics, warning projection, and failure-class preservation. Release gates: `npm run contract:check`, `npm run schema:coverage`, `npm test`, and `npm run perfect-spec:check -- --gate=publish`.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, npm publish blocking, package smoke, tracked dist output, README release examples, release workflow version matching, and GitHub Release tarball smoke. Release gates: `npm run version:check`, `npm run build`, `npm run pack:smoke`, `npm run remediation:check`, and release workflow tarball smoke for `officegen-v3.1.4.tgz`.

## Review 3: Asset / Office Workflow

Reviewed that PPTX/PDF view artifacts, internal CJK raster previews, proof-mode behavior, visual verify quality gates, and semantic inspect/diff behavior were not regressed by the envelope patch. Release gates: full `npm test`, generated perfect-spec evidence, and packaged tarball smoke.

## Review 4: UX / Critique / Traceability

Reviewed agent ergonomics for progressive disclosure, next action narrowing hints, artifact usability signal clarity, generated acceptance evidence, and release traceability. Release gates: review matrix, evidence manifest, and release checklist.
