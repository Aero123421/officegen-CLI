# officegen v5.2.0 Remediation Matrix

This matrix records the v5.2.0 hardening work after multi-agent CLI audits with Cursor Agent, Gemini CLI smoke attempts, opencode Kimi K2.6, and Codex subagent/code review.

## Review 1: Agent Contract / False Success

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |
| V52-001 | Template fill | Replacements could write an artifact while reporting `changed:false` and `mutationStatus:noop`. | fixed | `package_diff_changed` drives template mutation reporting and `tests/xlsx_template_v5.rs` verifies changed artifacts. |
| V52-002 | Diff | Non-mutating `diff` reported top-level `mutationStatus:changed`. | fixed | `tests/cli_contract_v51.rs` verifies changed diffs stay `not_applicable`. |
| V52-003 | JSON budget | Truncating a failed response could make it look like partial success. | fixed | `tests/cli_contract_v51.rs` verifies budgeted unknown commands remain `ok:false` and `readiness:blocked`. |
| V52-004 | Schema validate | `schema validate --schema-id` was ignored and envelope schema was used instead. | fixed | `tests/cli_contract_v51.rs` validates IR v2 with `--schema-id officegen.ir.document@2.0`. |
| V52-005 | Error category | Format-prefixed edit mismatches returned an unsupported code with runtime category. | fixed | `FORMAT_UNSUPPORTED` is categorized as unsupported and covered by unit tests. |
| V52-006 | PDF edit | Portable PDF edit could appear mutable despite unsupported writer fidelity. | fixed | `apply_pdf_edit_ops` fails closed and `tests/cli_contract_v51.rs` verifies no output artifact. |
