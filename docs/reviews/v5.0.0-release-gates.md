# Officegen-CLI v5.0.0 Release Gates

This checklist is the publish blocker for v5.0.0. Do not create the v5.0.0 tag, upload assets, or announce the release until the required gates pass for the exact release commit and version.

## Version Gate

Run the version script; do not edit version files by hand:

```bash
npm run version:bump -- 5.0.0
npm run version:check
```

## Native And Installer Gates

Build the native binary first, then run platform-appropriate smoke checks and the host v5 acceptance suite:

```bash
cargo build --release --locked
npm run installer:smoke

# macOS/Linux host
npm run native:smoke -- --bin target/release/officegen --expected-version 5.0.0
npm run v5:acceptance -- --bin target/release/officegen --expected-version 5.0.0

# Windows host
npm run native:smoke -- --bin target/release/officegen.exe --expected-version 5.0.0
npm run v5:acceptance -- --bin target/release/officegen.exe --expected-version 5.0.0
```

GitHub Actions runs A1-A12 v5 acceptance on the host release binary and preserves `.officegen/acceptance/v5.0.0/manifest.json` as release evidence. Matrix-built native assets must pass `native:smoke`; they are not required to rerun the full A1-A12 suite on every OS target.

Before uploading release assets:

```bash
npm run native:assets:check -- --version 5.0.0 --include-installers
```

## Repository Gates

These must pass before the release tag:

```bash
cargo fmt --check
cargo test --locked
cargo build --release --locked
npm run typecheck
npm test
npm run build
npm run v5:acceptance -- --bin target/release/officegen --expected-version 5.0.0
```

`npm run release:gate` is the local all-up gate for the common pre-tag path after the release binary has been built.

## Distribution Rule

The v5 distribution is the GitHub Release native binary plus checksum files and the `curl`/`irm` installers. Do not publish or upload an npm package/tarball for the supported CLI path. The historical TypeScript implementation lives under `legacy/typescript-v3-reference/` for reference only.
