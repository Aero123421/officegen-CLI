## Release Gates

Required before v3.1.2 tagging:

- `npm run version:check`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run pack:smoke`
- `npm run remediation:check`
- `npm run contract:check`
- `npm run schema:coverage`
- `npm run perfect-spec:evidence`
- `npm run perfect-spec:check -- --gate=publish`
- `npm run release-tarball:smoke -- dist/officegen-v3.1.2.tgz`

Post-tag release completion additionally requires fresh v3.1.2 GitHub install and release-tarball smoke evidence.

## Review 1: Agent Contract / False Success

Reviewed raster artifact quality, verify visual gating, runtime envelope readiness fields, selector lock evidence, objective failure projection, and false-success prevention. Release gates: `npm run contract:check`, `npm run schema:coverage`, `npm test`, and `npm run perfect-spec:check -- --gate=publish`.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, npm publish blocking, package smoke, tracked dist output, README release examples, release workflow version matching, and GitHub Release tarball smoke. Release gates: `npm run version:check`, `npm run build`, `npm run pack:smoke`, `npm run remediation:check`, and release workflow tarball smoke for `officegen-v3.1.2.tgz`.

## Review 3: Asset / Office Workflow

Reviewed PPTX PNG raster root cause, fallback behavior, visual verify quality gates, PPTX semantic inspect/diff/view behavior, and `pptx.setBold` selector application. Release gates: full `npm test`, generated perfect-spec evidence, and packaged tarball smoke.

## Review 4: UX / Critique / Traceability

Reviewed agent ergonomics for preview usability, paragraph-level semantic output, selector near-match hints, artifact usability warnings, generated acceptance evidence, and release traceability. Release gates: review matrix, evidence manifest, and release checklist.
