# v4.5.0 Review Checklist

Release focus: native Rust v4.5 contract truthfulness, MCP exclusion, compact capabilities, stable JSON envelopes, safety gates, and smoke evidence for the core Office/PDF command loop.

Review status values: `pass`, `warning`, `blocked`, `not-run`, `waived`.

## Required Release Commands

Run before tag creation:

```bash
npm run version:check
npm run installer:smoke
# macOS/Linux
npm run native:smoke -- --bin target/release/officegen --expected-version 4.5.0
# Windows
npm run native:smoke -- --bin target/release/officegen.exe --expected-version 4.5.0
cargo fmt --check
cargo test --locked
cargo build --release --locked
npm run typecheck
npm test
npm run build
npm run pack:smoke
```

Recommended evidence commands:

```bash
npm run contract:check
npm run schema:coverage
npm run remediation:check
npm run perfect-spec:evidence
npm run perfect-spec:check -- --gate=publish
```

## Review Questions

- Does every agent-visible command either work, fail closed, or clearly say it is policy-gated?
- Does the default runtime avoid MCP, native renderer execution, and external processes?
- Does `capabilities --agent --strict-json` stay concise while pointing to deeper schema/help/report commands?
- Do negative cases return non-zero exit codes and structured JSON with stable codes?
- Are all output paths inside the allowed project/output root unless policy explicitly allows otherwise?
- Are OOXML packages checked for ZIP risks before XML parsing?
- Does post-edit evidence include package diff, artifact path, and verify guidance?
- Does native renderer wording avoid implying proof when Office/LibreOffice was not run?
- Are v5 items deferred because they are out of v4.5 scope, not because failures were hidden?

## Waiver Rule

A required `V45-*` item can be waived only when all of the following are true:

- The release note or README-facing contract discloses the limitation.
- The CLI returns a structured blocked/unsupported/warning result instead of false success.
- The review table records the waiver reason and a v5 roadmap ID.
- The waiver does not weaken version sync, path safety, ZIP safety, installer, or native binary smoke.
