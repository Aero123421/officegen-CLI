# officegen v2.6.1 remediation matrix

Status values: `fixed`, `limited-disclosed`, `test-covered`, `deferred-v3`, `not-applicable`.

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |

| V26-K002 | View | PDF view was SVG/HTML placeholder-oriented instead of real page images | fixed | PDF inputs are rasterized through PDF.js and `@napi-rs/canvas`; tests assert PNG magic bytes and dimensions. |
| V26-K003 | View | Raster page artifacts could leak binary bytes into JSON envelopes | fixed | CLI artifact reports strip page `bytes` and `content` when writing view output directories. |

| V26-K005 | View | Raster dimensions were missing from page records | fixed | Raster pages now carry width, height, renderer, format, and stable page ids in the manifest. |

| V26-K007 | View | Packaged installs could miss raster dependencies unnoticed | test-covered | `pack:smoke` now runs PDF render plus `view --format png` and checks the PNG signature. |

## Review 1: Agent Contract / False Success

Reviewed raster view readiness, natural-language planning limits, JSON EditOps pass-through, and lock-owner parsing. Unsupported broad planning still reports limitations instead of pretending success.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, runtime dependency declarations, pack smoke coverage for PNG rendering, audit output, and required release gates for v2.6.1.

## Review 3: Asset / Office Workflow

Reviewed `view --format png/jpeg`, PDF raster artifacts, `prepare --format png`, Office native-renderer fallback behavior, manifest metadata, and contact sheet rendering.

## Review 4: UX / Critique / Traceability

Reviewed CLI behavior for `lock --agent`, new `--owner`, renderer capability visibility, plan outputs for Japanese goals, and artifact reports that are safe for JSON envelopes.
