# officegen v5.1.0 Remediation Matrix

This matrix records the v5.1.0 hardening work required before publishing the native Rust release.

## Review 1: Agent Contract / False Success

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |
| V51-001 | Edit dry-run | `edit --dry-run` required `--out` even though it writes nothing. | fixed | Dry-run output is optional and `tests/xlsx_template_v5.rs` covers no-artifact dry-run. |
| V51-002 | Edit schema | `officegen.edit.ops@1.2` rejected natural `ops` payloads. | fixed | `schemas/edit-ops-1.2.schema.json` allows `operations` or `ops`; runtime rejects conflicting arrays. |
| V51-003 | Error catalog | Emitted stable error codes were missing from `errors list`. | fixed | `tests/cli_contract_v51.rs` checks `OUTPUT_REQUIRED`, `SELECTOR_NOT_FOUND`, and `WORKFLOW_STEP_FAILED`. |
| V51-004 | Schema fetch | `schema fetch --out` returned success without writing the file. | fixed | `tests/cli_contract_v51.rs` verifies raw schema artifact creation. |
| V51-005 | JSON budget | `--json-budget-bytes` did not reduce large inspect payloads. | fixed | `tests/pptx_docx_v5.rs` verifies partial envelopes and bounded JSON. |
| V51-006 | Object map limit | `--object-map-limit` did not affect inspect output. | fixed | `tests/pptx_docx_v5.rs` verifies returned counts and truncation metadata. |
| V51-007 | Workflow failure | Failed workflows mixed trace artifacts with successful mutation semantics. | fixed | `tests/workflow_v5.rs` verifies failed workflows report `changed:false` and completed step count accurately. |

## Review 2: Install / Release / Packaging

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |

| V51-012 | Release gate | `release:gate` pinned v5.0.0 in package scripts. | fixed | `package.json` uses `--expected-version current` for local release gates. |
| V51-013 | Legacy npm path | npm CLI distribution remains removed. | not-applicable | Root `package.json` stays private release tooling with no `bin` path. |
