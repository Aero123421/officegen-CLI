# v2.2.0 Remediation Matrix

Status values: `implemented`, `limited-but-disclosed`, `optional-gated`, `benchmark-covered`.

| Review item | Status | v2.2.0 handling |
| --- | --- | --- |
| GitHub direct install fails because prepare needs missing TypeScript/dist | implemented | `prepare` now uses existing dist when present and errors clearly only when dist and TypeScript are both missing; `github-install:smoke` is a release gate. |
| Missing output artifact is hidden behind `ok:true` | implemented | Mutating template fills validate artifact existence and error details carry `artifacts[].exists:false`; envelope preserves artifacts. |
| JSON budget truncation drops artifacts | implemented | Progressive disclosure preserves artifacts and emits `result.partialSummary`. |
| Absolute `--out` recovery is too generic | implemented | `SECURITY_ABSOLUTE_OUT_DENIED` returns concrete relative `--out` examples, recommended fix, expected result. |
| Template title/date mis-inference | implemented | Field inference uses confidence and contextual placeholder kinds; generic titles remain `string`. |
| `pic` placeholder inferred as string | implemented | Template schema candidates support `image`; PowerPoint picture placeholders map to `image`. |
| Chart/table/image bindings use text ops | implemented | Template fill converts `chartData`, `image`, and `table` field types to dedicated edit operations. |
| Nested `templateMapSuggested` mapping collapses to `[object Object]` | implemented | Template map flattening accepts persisted nested suggestions and normalizes to field bindings. |
| `template fill` lacks validation-only pass | implemented | `template fill --validate-only` reports value, selector, binding, and artifact feasibility without writing. |
| Template selectors are not edit-resolver verified | implemented | Candidates/create/fill surface `editable`, `editableOps`, selector metadata and unsupported bindings fail atomically. |
| `design capture` tokens empty | implemented | Capture synthesizes non-empty color, typography, spacing, layout, and object style tokens from PPTX signals. |
| `design apply` overclaims design transfer | implemented | `--strategy theme-only|inspired|faithful` applies theme tokens plus captured bbox/font/image patterns for PPTX, runs dense-text fitting, preserves masters/placeholders, and reports `visualEffect`, `tokensApplied`, `changedParts`, `limitations`. |
| Design/layout unsupported formats look successful | limited-but-disclosed | Result contracts include `supported:false`, `noopReason`, and `formatCapabilities` for non-mutating/unsupported paths. |
| Chart stableObjectId mismatch across inspect/view/template/design/edit | implemented | Design extraction now mirrors PPTX object ID derivation for chart frames where shape ids are available. |
| `verify` score goes to zero from repeated warnings | implemented | Warning aggregation, capped penalties, `warningSummary`, `topRisks`, `scoreBreakdown`, and recommended repairs are returned. |
| `TEXT_OVERFLOW_RISK` lacks focused top risks | implemented | Verify surfaces top repeated risks and top five repair recommendations. |
| Diff misses chart XML/theme/table/media changes | implemented | Diff includes semantic OOXML part changes for charts, embedded workbook, theme, table, and media paths. |
| PDF inspect with textBlocks 0 is too quiet | implemented | Zero text blocks produce a quality warning and points agents to page preview artifacts/native PDF tooling. |
| Heavyweight local text recognition availability is environment-dependent | implemented | Local text-recognition tooling was removed from v2.2.0; scanned PDFs are handled through page preview artifacts for AI vision review. |
| Native Office-to-PDF/repair dialog detection is environment-dependent | implemented | `renderer doctor` discovers LibreOffice headless and Windows Office COM backends; `export --mode native` and `verify --native` use trusted renderers when policy allows and otherwise fail with explicit diagnostics. |
| XLSX summary shallow for formulas/tables/pivots/slicers | implemented | XLSX inspect returns a workbook map with formulas, input cells, validations, hidden/protected sheets, named ranges, links, tables/charts/pivots/slicers. |
| Need scoped XLSX inspect to avoid huge summaries | implemented | `inspect xlsx --sheet <name> --range A1:K40` is supported. |
| XLSX verify needs formula/name/link/protection flags | implemented | `verify xlsx --formulas --named-ranges --external-links --protected-sheets` reports scoped risks. |
| Excel pivot/slicer/chart/table operation claims need real guards | implemented | Table and chart OOXML edits are first-class; pivot definitions can be marked `refreshOnLoad`, slicers support selection updates, and Excel COM refresh remains an optional native verification backend. |
| DOCX structure map shallow | implemented | `inspect docx --structure` adds headings, sections, headers/footers, tables, fields, content controls, comments, redlines, placeholders. |
| Word redline/style/header/footer operations shallow | implemented | DOCX edit supports tracked insert/delete/replace, comment ranges, style application, and header/footer mutation through OOXML with structure-map verification. |
| Public benchmark corpus should not commit Office binaries | benchmark-covered | `benchmarks/office-corpus/manifest.json`, `benchmark:fetch`, and `benchmark:review` manage external corpus artifacts under `.officegen`. |
| Corpus fetch failures need explicit fallback/failure records | benchmark-covered | Fetch result records source URL, fallback URL, hash, bytes, and failure reason. |
| Capabilities/help over-advertise plan-only/optional features | limited-but-disclosed | Capabilities include supported formats, format capabilities, planOnly/mutatesOffice, optional native requirements, and known limitations. |
| Top 10 / 10/10 native fidelity goals exceed portable CLI guarantees | implemented | Native Office COM and LibreOffice renderer discovery/export/verify are implemented through `renderer doctor`, `export --mode native`, and `verify --native`; unavailable renderers fail with explicit diagnostics. |
