# officegen v2.6.0 remediation matrix

Status values: `fixed`, `limited-disclosed`, `test-covered`, `deferred-v3`, `not-applicable`.

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |
| V26-A001 | Goal Contract | The complete Office operation OS target was not machine-checkable | fixed | `/goal` now defines the v2.6.0 Goal Contract, acceptance suite, schemas, and completion/evidence checklists. |
| V26-A002 | Goal Contract | Goal drift could pass review without evidence | test-covered | `npm run goal:check` validates 10 domains, 31 requirements, and 28 acceptance tests. |
| V26-B001 | CLI Surface | `prepare` existed only as `run prepare-reference` | fixed | `prepare` is now a top-level feature/command and reuses the reference preparation workflow. |
| V26-B002 | CLI Surface | Command/capability contracts omitted the prepare workflow | fixed | Core feature names, capabilities, metadata, and help examples include `prepare` and `prepare reference`. |
| V26-C001 | Manifest | Artifact manifests could verify zero artifacts | fixed | `manifest --out` now creates view artifacts and includes source, page previews, object-map, contact-sheet, hashes, and artifact records. |
| V26-C002 | Manifest | Artifact records lacked stable ids and sha256 prefixes | fixed | `artifactRecord` now adds `artifactId`, `role`, prefixed `sha256`, and optional agent attribution. |
| V26-C003 | View | View manifests lacked object-map hash/provenance fields | fixed | `writeViewArtifacts` records page artifact ids, renderer, fidelity, coordinate system, and object-map hash. |
| V26-D001 | Selector | Low-confidence selector matches were not enforceable | fixed | EditOps options include `minSelectorConfidence`; selector validation blocks low-confidence matches. |
| V26-D002 | Selector | PDF edits ignored stale-plan hashes | fixed | PDF edit results now carry input/object-map hashes and reject mismatched expected hashes. |
| V26-D003 | Selector | DOCX/XLSX/PDF natural selector fields were under-modeled | fixed | Selector schema and resolver now support source/xml path, page, story, paragraph, table, row, column, range, relationship, asset, comment, and revision hints. |
| V26-E001 | Collaboration | Lock scope was recorded but not enforced | fixed | `edit --lock` now checks operations against slide/sheet/paragraph/range/stableObjectId scopes. |
| V26-E002 | Collaboration | Edit attribution did not flow into transactions | fixed | Edit results and transaction JSON now include agent, command, operation count, ops hash, lock path, and transaction history JSONL. |
| V26-F001 | PPTX | High-level deck operations were missing | fixed | Added schema/dispatch support for addSlideFromLayout, replaceBodyBullets, formatAllTitles, fitContentToPlaceholder, align/distribute, alt text, and speaker notes. |
| V26-F002 | PPTX | Accessibility metadata was not editable through EditOps | fixed | `pptx.setAltText` updates `p:cNvPr` title/description metadata for shapes, pictures, and charts. |
| V26-F003 | PPTX | Speaker notes were outside EditOps | fixed | `pptx.setSpeakerNotes` creates/updates notes slide parts, content type overrides, and slide relationships. |
| V26-G001 | DOCX | Object map was paragraph-only | fixed | DOCX inspect now adds table, tableCell, and image entries with story/source/relationship hints. |
| V26-G002 | DOCX | Run-split replacement lacked a safer operation | fixed | `docx.replaceTextSmart` rewrites a selected paragraph through a joined text view while preserving the paragraph container. |
| V26-G003 | DOCX | Table cells were not first-class edit targets | fixed | `docx.setTableCellText` edits selected DOCX table cell text. |
| V26-H001 | XLSX | Named ranges and validations were not object-map entries | fixed | XLSX inspect now adds namedRange and validation objects with selector hints. |
| V26-H002 | XLSX | Formula writes lacked a safety gate | fixed | `xlsx.setFormula` now blocks external, volatile, indirection, and link-capable formulas by default and invalidates calcChain. |
| V26-H003 | XLSX | Defined names were inspect-only | fixed | Added `xlsx.definedName.set` and `xlsx.definedName.delete` EditOps. |
| V26-I001 | Diff | `diff --visual --out` did not emit review artifacts | fixed | CLI diff now writes diff JSON, before/after view artifacts, and a diff artifact manifest. |
| V26-I002 | Verify | Gates were ad hoc instead of schema-backed | fixed | Added strict `officegen.verify.gates@1.2` schema and CLI validation now uses the schema registry. |
| V26-I003 | Prepare | Recommended gates file was referenced but not generated | fixed | `prepare`/`run prepare-reference` now writes `gates.json` and includes it in artifacts. |
| V26-J001 | Release | v2.6.0 release matrix was missing | fixed | This matrix classifies the v2.6.0 implementation/remediation work. |
| V26-J002 | Scope | Native high-fidelity raster rendering still depends on environment renderers | limited-disclosed | v2.6.0 strengthens provenance/artifacts and keeps native renderer requirements explicit rather than pretending an unavailable renderer exists. |
| V26-J003 | Scope | Full replacement for Microsoft Office layout engines remains out of repo scope | deferred-v3 | The Goal Contract treats external/native renderers as pluggable evidence providers instead of reimplementing Office layout from scratch. |
| V26-J004 | Scope | OCR-first document understanding was requested by neither workflow nor acceptance contract | not-applicable | v2.6.0 focuses render/object-map/EditOps/verify workflows; scanned PDF handling remains a preview/native-renderer concern. |

## Review 1: Agent Contract / False Success

Reviewed Goal Contract coverage, selector confidence, stale-plan behavior, manifest verification, and readiness/objective semantics. Unsupported intents must still block or disclose limitations rather than reporting false success.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, release matrix presence, goal checker packaging, command contract coverage, schema coverage, and required AGENTS release gates. v2.6.0 release must still pass build, pack smoke, and GitHub/release tarball smoke after commit/tag.

## Review 3: Asset / Office Workflow

Reviewed prepare/manifest/view artifact workflows, PPTX notes/accessibility/image/chart paths, DOCX table/image object maps, XLSX named ranges/formula safety, and diff/verify sidecar artifacts.

## Review 4: UX / Critique / Traceability

Reviewed top-level prepare help, artifact records, scoped locks, transaction attribution/history, `/goal` completion checklists, and this remediation matrix. v2.6.0 moves officegen toward an AI Office operation OS substrate with machine-checkable evidence.
