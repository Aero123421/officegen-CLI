# v2.3.0 Review Remediation Matrix

Status values: `implemented`, `limited-but-disclosed`, `optional-gated`, `benchmark-covered`.

| Review item | Status | v2.3.0 response |
| --- | --- | --- |
| GitHub direct install looks successful but misses `packages/cli/dist/main.js` | implemented | Packaging now whitelists runtime dist files, `prepare` remains dist-aware, and `github-install:smoke` verifies installed dist entrypoints and CLI execution. `github-install:remote-smoke` is available for tag/release gates. |
| Agent logs suffer PowerShell mojibake and shell transcript ambiguity | implemented | `officegen run` supports `--log-jsonl`, `--manifest`, `--summary`, and `--strict-json` for UTF-8 CLI-owned evidence. |
| Agent final artifact claims are unreliable | implemented | Run manifests record artifact path, exists, bytes, sha256, creator command, input, missing expected artifacts, and unexpected output-root artifacts. |
| `schema fetch` natural command fails | implemented | `schema fetch <id>` is an official alias for `schema get <id>` and appears in capabilities/help. |
| Failed logs can contain exit code 0 or success-looking output | implemented | `officegen run` records `exitCode`, `envelopeOk`, `ok`, and failed step details in JSONL/manifest. |
| `inspect/verify --out` report semantics are confusing | implemented | `--report-out` writes report JSON for report-style commands; `verify --out` remains compatibility behavior. |
| Output scope is weak and stray outputs are hard to catch | implemented | `officegen run --output-root --deny-outside-output-root --expected-artifacts` detects missing/out-of-root artifacts. |
| Large JSON output needs narrowing guidance | implemented | Progressive disclosure now includes `recommendedNarrowCommands`; CLI accepts `--object-map-limit`, `--fields`, `--sheet`, `--range`, `--slides`, and `--pages`. |
| `template fill --validate-only` returns false positives | implemented | Validate-only now dry-runs the real edit resolver and fails with `TEMPLATE_VALIDATE_FAILED` when selectors cannot be resolved. |
| Template `editable:true` mismatches real edit resolver | implemented | Validate-only enriches bindings with resolver status, match count, and unsupported reason. |
| Large XLSX verify can hang | implemented | `verify --timeout-ms` returns partial results and `VERIFY_TIMEOUT` warnings by phase. |
| `NATIVE_RENDERER_NOT_RUN` harms quality score as document warning | implemented | Native renderer absence is categorized as `environment`, producing `pass_with_environment_gap` when it is the only warning. |
| Repeated `TEXT_OVERFLOW_RISK` lacks focused repair priority | implemented | Verify surfaces worst overflow risks with slide/page/object metadata and repair guidance. |
| Low-reasoning agents need business-quality lint | implemented | New `critique` and dry-run `improve` commands flag missing charts/assets, high text density, repeated headings, and dashboard gaps. |
| Benchmark regression tracking should be first-class | benchmark-covered | New `benchmark run` and `benchmark compare` commands produce versioned corpus reports without committing Office binaries. |
| Direct install should not be over-recommended while release tarball is safer | limited-but-disclosed | README presents release tarball as the recommended deterministic install and documents GitHub direct install as smoke-tested. |
| Native renderer/OCR expectations should remain honest | optional-gated | OCR is not reintroduced; native renderer behavior remains optional-gated and explicitly classified as environment evidence. |
