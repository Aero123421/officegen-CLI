## Release Gates

Required before v3.0.0 tagging:

- `npm run version:check`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run pack:smoke`
- `npm run remediation:check`

Version bump must be done with `npm run version:bump -- 3.0.0` before the final release commit/tag.

## Review 1: Agent Contract / False Success

Reviewed agent-visible capability payloads for complete-editing overclaims. Release gates: `npm run typecheck`, `npm test`, `npm run remediation:check`.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, README release examples, package smoke coverage, and post-tag install smoke expectations. Release gates: `npm run version:check`, `npm run build`, `npm run pack:smoke`, and post-tag GitHub install smoke commands.

## Review 3: Asset / Office Workflow

Reviewed EditOps schema and Office workflow boundaries for chart, pivot, slicer, asset replacement, native renderer, DOCX, and PDF overlay behavior. Release gates: schema tests in `packages/core/src/schemas.test.ts` and full `npm test`.

## Review 4: UX / Critique / Traceability

Reviewed README feature claims, agent-facing capability disclosures, critique/verify limitation language, and visible remediation evidence. Release gates: review matrix plus release checklist.
