# v4.0.0 Remediation Matrix

Release focus: native Rust runtime, GitHub Release binary assets, macOS/Linux `curl` installer, Windows `irm` installer, and explicit disclosure for parity gaps that remain outside the Rust core.

## Review 1: Agent Contract / False Success

| ID | Area | Finding | Status | Evidence |
| --- | --- | --- | --- | --- |
| V40-001 | Envelope | Rust CLI must keep `officegen.envelope@1.2` and runtime v2 top-level fields. | fixed | `src/runtime.rs` command envelope tests assert schema, runtimeEnvelope, objective fields, and capabilities hash. |
| V40-002 | Strict JSON | Agent mode must emit structured JSON for success and failure. | test-covered | `unknown_command_envelope_has_structured_error` covers structured failure envelope and command suggestions. |
| V40-003 | Runtime dependency | v4 runtime must not require Node.js when installed from native assets. | fixed | Rust binary reports `runtime.kind=rust-native` and `nodeRequired=false`; native smoke runs the binary directly. |
| V40-004 | Placeholder commands | Commands not fully ported must not claim mutation success. | fixed | `export` rejects unsupported conversions and deferred mutation-heavy surfaces return `FEATURE_NOT_IMPLEMENTED`. |
| V40-005 | Schema compatibility | Existing agent-facing schema IDs must remain discoverable. | limited-disclosed | `schema list/get` exposes core v3 IDs while nested schemas remain permissive during Rust migration. |
| V40-006 | Capabilities honesty | Capabilities must disclose limited Office/PDF fidelity. | fixed | Rust capabilities include `featureContracts`, `formatCapabilities`, and `unsupportedNow` limitations. |
