## Release Gates

Required before v3.1.0 tagging:

- `npm run version:check`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run pack:smoke`
- `npm run remediation:check`
- `npm run contract:check`
- `npm run schema:coverage`
- `npm run perfect-spec:evidence`
- `npm run perfect-spec:check`

Post-tag release completion additionally requires fresh v3.1.0 GitHub install and release-tarball smoke evidence.

## Review 1: Agent Contract / False Success

Reviewed runtime envelope truthfulness, false-success blocking, command-specific options, compact schema diagnostics, selector locks, and capability profiles. Release gates: `npm run contract:check`, `npm run schema:coverage`, `npm test`, and `npm run perfect-spec:check`.

## Review 2: Install / Release / Packaging

Reviewed version synchronization, package smoke, tracked dist output, README release examples, release workflow version matching, and post-tag install smoke expectations. Release gates: `npm run version:check`, `npm run build`, `npm run pack:smoke`, and post-tag GitHub install smoke commands.

## Review 3: Asset / Office Workflow

Reviewed OOXML ZIP safety, XLSX/PPTX/PDF bug fixes, object graph/selector resolution, object crop evidence, repair plans, and mutating edit output warnings. Release gates: full `npm test`, `npm run goal:check`, and generated perfect-spec evidence.

## Review 4: UX / Critique / Traceability

Reviewed agent ergonomics, progressive disclosure, help/capabilities examples, runtime evidence manifests, run trace append behavior, and replay fields. Release gates: review matrix, evidence manifest, and release checklist.
